# OD-Recruit 招聘管理系统 - 业务需求文档
> 版本: 3.4  
> 日期: 2026-05-21  
> 定位: 纯业务功能描述（面向产品/用户/业务人员）

---

## 1. 产品概述

### 1.1 产品定位
本系统为企业级招聘全生命周期过程管理工具，追踪从候选人"简历进入"到"正式入职/离职"的完整招聘流程。

**核心业务价值：**
- 标准化招聘各环节的流程
- 沉淀候选人全流程数据
- 识别招聘流程卡点，优化招聘效率
- 基于角色的访问控制，保障数据安全

### 1.2 系统使用角色

| 业务角色 | 业务职责 |
|----------|----------|
| **招聘顾问（Consultant）** | 候选人全流程操作：录入候选人、推进阶段、面推、录入机考/韧测/面试数据 |
| **招聘主管（Manager）** | 含顾问全部权限，可配置系统参数：业务线/阶段/机考试卷/角色/权限分配 |
| **高管/HRBP** | 查看招聘统计报表与效率分析数据 |
| **系统管理员** | 权限配置、账号管理 |

## 2. 核心业务流程（15个阶段）

```
候选录入
    ↓
机考申报 → 机考完成
    ↓
韧测申报 → 韧测完成
    ↓
推荐面试
    ↓
资面安排
    ↓
技术面试一 → 技术面试二
    ↓
主管面试
    ↓
租用审批
    ↓
Offer
    ↓
待入职
    ↓
入职
    ↓
离职
```

**阶段推进条件业务规则：**

| 当前阶段 | 下一阶段 | 触发条件 |
|----------|----------|----------|
| 候选录入 | 机考申报 | 自动进入下阶段 |
| 机考申报 | 机考完成 | 机考完成日期录入系统 |
| 机考完成 | 韧测申报 | 自动进入下阶段 |
| 韧测申报 | 韧测完成 | 韧测完成日期录入系统 |
| 韧测完成 | 推荐面试 | 顾问点击【面推】按钮 |
| 推荐面试 | 资面安排 | 面试筛选结论录入（非待筛选） |
| 资面安排 | 技术面试一 | 资格面试结论 = 通过 |
| 技术面试一 | 技术面试二 | 技术面试一结论 = 通过 |
| 技术面试二 | 主管面试 | 技术面试二结论 = 通过 |
| 主管面试 | 租用审批 | 主管面试结论 = 通过 |
| 租用审批 | Offer | 审批结论 = 通过 |
| Offer | 待入职 | Offer录入通过 + 顾问点击【推进】按钮 |
| 待入职 | 入职 | 员工管理页面录入实际入职日期 |
| 入职 | 离职 | 录入离职日期+离职类型 |

## 4. 招聘效率分析业务规则

### 4.1 阶段停留时间统计

**业务目标：**
> 追踪各阶段停留时长，自动识别流程卡点，优化招聘整体效率。

### 4.2 时间线数据采集
每一个候选人每进入一个阶段，都需要精确采集：
- **进入时间点**：候选人实际流入该阶段的时间
- **离开时间点**：候选人实际流出该阶段的时间
- **停留时长**：自动计算该候选人本阶段停留时间（单位：小时/天）

### 4.3 推荐面试阶段特殊规则

推荐面试阶段的"离开时间"**不由系统自动计算**，需与业务人员在界面录入的**反馈日期**对齐：
```
离开时间 = 业务人员录入反馈日期 + 18:00（固定下班时点）
```

### 4.4 流程卡点自动判定规则

通过数据自动识别哪个阶段是招聘效率的瓶颈：

```
全局平均停留时长 = 所有已完成阶段样本总时长 / 总样本数

IF 某一阶段平均停留时长 ≥ 全局平均 × 150% THEN
    该阶段自动标记为「卡点预警」
    统计页高亮展示提示
ENDIF
```

### 4.5 统计页查询能力

| 查询维度 | 业务查询选项 |
|----------|--------------|
| 时间周期筛选 | 指定开始日期/结束日期范围查询 |
| 单阶段筛选 | 可只看单个阶段的停留数据 |
| 百分位统计 | 中位数/P50/P75/P90 多角度看分布（避免均值被极端值影响） |
| 明细列表 | 候选人+阶段级别的全量停留时间明细表格 |
| 可视化 | 柱状图直观对比各阶段平均停留时长（卡点红色预警） |

---

## 5. 阶段回退业务规则

支持业务操作失误时将候选人**退回上一阶段**重新操作，时间线同步更新规则：

```
同一个候选人 同一个阶段 只有一条时间线记录（不新增）
  → 回退重入该阶段：
    - 更新「进入时间」为回退时点
    - 重置「离开时间」为空（重新开始计时）
```
