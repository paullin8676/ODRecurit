# 面试管理增加候选人通过/不通过筛选功能

## 需求分析

### 筛选逻辑
- **不通过**：候选人任一面试阶段有任一阶段 `passed = false`，则该候选人为不通过
- **通过**：其他情况均认为是通过（包括 pending 和 passed）

### 需要实现的功能
1. 在面试管理前端页面添加"通过/不通过"筛选下拉框
2. 后端 API 支持根据面试轮次的 passed 状态筛选候选人

## 技术分析

### 前端分析
- 文件：`frontend/src/views/InterviewStage.vue`
- 现有筛选：`searchForm` 包含 `name` 和 `currentStage`
- 需要新增：`passStatus` 字段（pass/fail/all）

### 后端分析
- 文件：`backend/src/routes/interview.js`
- 当前筛选逻辑：基于 `Interview.currentStage` 和候选人姓名
- 需要新增：
  - 解析 `passStatus` 参数
  - 根据面试轮次的 `passed` 状态判断候选人是否通过
  - 筛选出包含 `passed=false` 轮次的候选人为不通过

### 数据结构
```
Interview
  - currentStage: 当前阶段
  - finalStatus: pending/passed/failed
  - rounds: [InterviewRound]
    - passed: null/true/false

Candidate
  - currentStage: 候选人当前阶段
```

## 实现步骤

### 步骤 1: 前端 - 添加筛选表单
- 在 `searchForm` 中添加 `passStatus` 字段
- 添加筛选下拉框（全部/通过/不通过）
- 绑定 `@change` 事件触发搜索

### 步骤 2: 前端 - 修改 fetchEmployees
- 在 API 参数中添加 `passStatus`
- 传递 `passStatus` 给后端

### 步骤 3: 后端 - 修改 GET / 接口
- 接收 `passStatus` 参数
- 添加筛选逻辑：
  - `fail`：筛选有任一 `passed = false` 轮次的候选人
  - `pass`：筛选没有 `passed = false` 轮次的候选人
  - `all`：不筛选

### 步骤 4: 验证和测试
- 验证筛选逻辑是否正确
- 验证分页是否正常工作

## 注意事项
- 需要在 `transformInterview` 后进行筛选，因为需要使用 `rounds` 数据
- 筛选是基于面试轮次的 `passed` 状态，不是 `finalStatus`
