# 计划：删除面试模块产品线强关联

## 需求概述
1. 删除面推弹框中的产品线和产品负责人，只保留候选人和推荐日期
2. 推荐日期可编辑，只能选择今天和今天之后的日期
3. 删除候选人和产品线的强关联
4. 面试管理中点击编辑，产品线依旧保持在推荐面试区块，改为可空且非必填字段
5. **一个候选人只能有一次面试机会**（新增约束）

## 数据库设计调整

### 当前设计问题
- `Interview` 表通过 `candidateProductLineId` 关联到 `CandidateProductLine`
- `CandidateProductLine` 有唯一约束 `UNIQUE(candidateId, productLineId)`，限制一个候选人只能有一个产品线记录
- `CandidateProductLine.productLineId` 是 NOT NULL

### 新数据库设计

#### 1. Interview 表修改
```sql
-- 直接关联 Candidate，添加 candidateId 字段（NOT NULL + UNIQUE）
-- candidateProductLineId 改为可空（兼容旧数据）

CREATE TABLE IF NOT EXISTS Interview (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    candidateId INTEGER NOT NULL UNIQUE,  -- 新增：直接关联候选人，唯一约束保证一个候选人只能有一次面试
    candidateProductLineId INTEGER,       -- 改为可空，兼容旧数据
    currentStage VARCHAR(50) NOT NULL DEFAULT 'recommend_interview',
    finalStatus VARCHAR(20) NOT NULL DEFAULT 'pending',
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (candidateId) REFERENCES Candidate(id),
    FOREIGN KEY (candidateProductLineId) REFERENCES CandidateProductLine(id)
);
```

**关键变更**：
- `candidateId` 字段为 NOT NULL + UNIQUE，确保一个候选人只能有一条面试记录
- `candidateProductLineId` 改为可空，不再强制要求关联产品线

#### 2. CandidateProductLine 表修改
```sql
-- productLineId 改为可空
-- 移除 UNIQUE(candidateId, productLineId) 约束

CREATE TABLE IF NOT EXISTS CandidateProductLine (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    candidateId INTEGER NOT NULL,
    productLineId INTEGER,               -- 改为可空
    recommendDate DATETIME,
    interviewStage VARCHAR(50) DEFAULT 'recommend_interview',
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (candidateId) REFERENCES Candidate(id),
    FOREIGN KEY (productLineId) REFERENCES ProductLine(id)
);
```

#### 3. 索引调整
```sql
-- 添加索引
CREATE INDEX IF NOT EXISTS idx_interview_candidateId ON Interview(candidateId);
```
