# 面试阶段推进判断逻辑修改计划

## 一、需求分析

### 1.1 推进判断条件（表格视图）
每个阶段显示推进按钮的条件：
- **必填项必须有值**（当前阶段要求的字段）
- **当前阶段的 interview_round.current_status = 'passed'**
- 面试状态不为 'failed'

### 1.2 对话框中的"保存&推进"逻辑
操作流程：
1. 先保存当前阶段的表单数据
2. 保存成功后，检查是否可以推进（按1.1的条件判断）
3. 如果**不能推进**：仅保存数据，停留在当前阶段，关闭对话框
4. 如果**可以推进**：保存数据后，自动推进到下一阶段，关闭对话框

### 1.3 阶段初始化规则
- **韧测完成 → 推荐面试**：需要初始化
  - `scheduledDate` = 当前日期
  - `currentStatus` = 'pending_filter'（待筛选）
- **其他阶段推进**：**无需初始化** `scheduledDate` 和 `currentStatus`
  - 让用户在编辑时自己填写

### 2.1 前端修改 ([InterviewStage.vue](file:///Users/paul/Documents/Opencode/OD-Recruit/frontend/src/views/InterviewStage.vue))

#### 2.1.1 修改 `canAdvance()` 函数（第995行）
**当前逻辑**：
- recommend_interview: status不为null且不为pending_filter
- 其他阶段: 有日期 AND status === 'passed'

**新逻辑**：
```javascript
const canAdvance = (row) => {
  const currentStage = row.currentStage
  
  // 如果面试状态为失败，不能推进
  if (row.currentStatus === 'failed') {
    return false
  }
  
  // pending_onboarding/entry/leave 不能推进
  if (['pending_onboarding', 'entry', 'leave'].includes(currentStage)) {
    return false
  }
  
  const currentRoundStatus = getRoundStatus(row, currentStage)
  
  // 必填项检查 + status === 'passed'
  switch (currentStage) {
    case 'recommend_interview':
      // 推荐面试：当前状态不为待筛选即可（不一定需要passed）
      return currentRoundStatus !== null && currentRoundStatus !== 'pending_filter'
    case 'qualification_interview':
    case 'tech_interview_1':
    case 'tech_interview_2':
    case 'manager_interview':
    case 'approval':
    case 'offer':
      // 这些阶段：必须有日期 AND currentStatus === 'passed'
      return getRoundDate(row, currentStage) && currentRoundStatus === 'passed'
    default:
      return false
  }
}
```

#### 2.1.2 修改 `canAdvanceInDialog()` 函数（第1030行）
**新逻辑**：与 `canAdvance()` 类似，但读取 `interviewForm.rounds` 的数据

```javascript
const canAdvanceInDialog = () => {
  if (!selectedInterview.value) {
    return false
  }
  
  // 如果面试状态为失败，不能推进
  if (selectedInterview.value.currentStatus === 'failed') {
    return false
  }
  
  const currentStage = selectedInterview.value.currentStage
  
  // pending_onboarding/entry/leave 不能推进
  if (['pending_onboarding', 'entry', 'leave'].includes(currentStage)) {
    return false
  }
  
  const formRound = interviewForm.rounds[currentStage]
  const formRoundStatus = formRound?.currentStatus
  
  // 必填项检查 + status === 'passed'
  switch (currentStage) {
    case 'recommend_interview':
      return formRoundStatus !== null && formRoundStatus !== 'pending_filter'
    case 'qualification_interview':
    case 'tech_interview_1':
    case 'tech_interview_2':
    case 'manager_interview':
    case 'approval':
    case 'offer':
      return formRound?.scheduledDate && formRoundStatus === 'passed'
    default:
      return false
  }
}
```

#### 2.1.3 修改 `handleSaveAndAdvance()` 函数（需要新增）
添加新的处理函数，实现"保存&推进"逻辑：

```javascript
const handleSaveAndAdvance = async () => {
  try {
    // 1. 先保存数据
    await handleSave()
    
    // 2. 检查是否可以推进
    if (!canAdvanceInDialog()) {
      ElMessage.info('当前阶段未完成，仅保存数据')
      return
    }
    
    // 3. 可以推进，执行推进操作
    const currentStage = selectedInterview.value.currentStage
    const currentStageData = interviewForm.rounds[currentStage] || {}
    
    await interviewApi.advance(selectedInterview.value.id, {
      currentStageData: {
        scheduledDate: currentStageData.scheduledDate,
        content: currentStageData.content,
        currentStatus: currentStageData.currentStatus,
        feedbackDate: currentStageData.feedbackDate
      }
    })
    
    ElMessage.success('保存并推进成功')
    dialogVisible.value = false
    fetchEmployees()
  } catch (error) {
    ElMessage.error('保存并推进失败')
  }
}
```
