# 候选人面试系统 - 产品需求文档

## 概述
- **摘要**：开发一个候选人面试系统，包含候选人基本信息、前置阶段数据、面试阶段数据，并在面试阶段进行产品线关联。
- **目的**：提供一个结构化的系统，管理候选人从录入到面试的全流程，确保面试过程的标准化和可追踪性。
- **目标用户**：招聘顾问、招聘主管、系统管理员。

## 目标
- 实现候选人基本信息的录入和管理
- 实现前置阶段数据的记录和管理
- 实现面试阶段数据的记录和管理
- 实现面试阶段的产品线关联
- 提供完整的候选人状态追踪

## 非目标（范围外）
- 候选人入职后的管理
- 薪资谈判和福利管理
- 外部招聘渠道的集成

## 背景与上下文
- 系统基于Vue 3和Express.js开发，使用SQLite数据库
- 已有的用户管理、产品线管理、试卷管理等模块
- 系统需要支持多产品线、多面试官的协作
- 数据库结构设计为三个主要表：
  - 候选人基本信息表：存储候选人的基本信息，每个候选人只有一条记录
  - 考试阶段表：存储候选人的机考和韧测信息，每个候选人只有一条记录
  - 面试阶段表：存储候选人的面试信息，每个候选人可以有多条记录，每条对应一个产品线

## 功能需求
- **FR-1**：候选人基本信息管理
  - 支持录入和编辑候选人的姓名、性别、手机号、邮箱、身份证号等基本信息
  - 支持查询和筛选候选人
  - 支持候选人的删除操作
  - 每个候选人只有一条基本信息记录，存储在候选人基本信息表中
  - 界面交互单独显示基本信息模块

- **FR-2**：前置阶段数据管理
  - 支持记录候选人的机考信息（试卷选择、考试日期、得分、是否通过等）
  - 支持记录候选人的韧测信息（测试类型、测试日期、得分、是否通过等）
  - 每个候选人只有一条前置阶段数据记录，存储在考试阶段表中
  - 界面交互单独显示前置阶段模块

- **FR-3**：面试阶段数据管理
  - 支持记录推荐面试信息（推荐日期、产品线选择等）
  - 支持记录资格面试信息（面试日期、面试官、结论、是否通过等）
  - 支持记录技术面试信息（面试日期、面试官、评价、是否通过等）
  - 支持记录主管面试信息（面试日期、面试官、评价、是否通过等）
  - 支持记录审批、Offer、入职、离职等信息
  - 每个候选人可以有多个面试阶段记录，每个对应一个产品线，存储在面试阶段表中
  - 即使候选人被某个产品线录取，其他产品线的面试记录仍需保留
  - 界面交互单独显示面试阶段模块，支持产品线切换

- **FR-4**：产品线关联管理
  - 在面试阶段（推荐面试）进行产品线关联
  - 支持一个候选人关联多个产品线
  - 支持产品线的切换和管理
  - 产品线关联仅在面试阶段表中实现
  - 即使候选人被某个产品线录取，其他产品线的面试记录仍需保留
  - 界面交互在面试阶段模块中实现产品线关联和切换

- **FR-5**：状态管理
  - 支持候选人在不同阶段的状态切换
  - 支持阶段的推进和回退
  - 支持状态的可视化展示

## 非功能需求
- **NFR-1**：性能
  - 系统响应时间不超过2秒
  - 支持至少1000个候选人的管理

- **NFR-2**：可用性
  - 系统可用性达到99%
  - 支持数据备份和恢复

- **NFR-3**：安全性
  - 支持用户认证和授权
  - 敏感数据加密存储
  - 操作日志记录

- **NFR-4**：可扩展性
  - 支持未来功能的扩展
  - 支持数据库迁移到MySQL

## 约束
- **技术**：
  - 前端：Vue 3 + Element Plus
  - 后端：Express.js + Sequelize
  - 数据库：SQLite（可迁移到MySQL）

- **业务**：
  - 产品线关联只能在面试阶段进行
  - 候选人状态流转遵循既定流程

- **依赖**：
  - 已有的用户管理模块
  - 已有的产品线管理模块
  - 已有的试卷管理模块

## 假设
- 系统用户已经完成注册和登录
- 产品线信息已经在系统中配置
- 试卷和测试类型信息已经在系统中配置

## 验收标准

### AC-1：候选人基本信息管理
- **Given**：系统已登录，用户有权限管理候选人
- **When**：用户录入候选人基本信息并保存
- **Then**：候选人信息被成功保存，显示在候选人列表中
- **Verification**：`programmatic`

### AC-2：前置阶段数据管理
- **Given**：候选人已录入基本信息
- **When**：用户录入机考和韧测信息并保存
- **Then**：前置阶段数据被成功保存，候选人状态更新
- **Verification**：`programmatic`

### AC-3：面试阶段数据管理
- **Given**：候选人已完成前置阶段
- **When**：用户录入面试阶段信息并保存
- **Then**：面试阶段数据被成功保存，候选人状态更新
- **Verification**：`programmatic`

### AC-4：产品线关联管理
- **Given**：候选人进入面试阶段
- **When**：用户为候选人关联产品线并保存
- **Then**：产品线关联成功，候选人与产品线建立关联关系
- **Verification**：`programmatic`

### AC-5：状态管理
- **Given**：候选人处于某个阶段
- **When**：用户推进或回退候选人状态
- **Then**：候选人状态成功更新，显示在系统中
- **Verification**：`programmatic`

### AC-6：系统集成
- **Given**：系统各模块已配置完成
- **When**：用户在不同模块间操作
- **Then**：数据在各模块间正确流转，无错误
- **Verification**：`human-judgment`
