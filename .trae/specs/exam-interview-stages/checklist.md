  - [ ] 表之间的关联关系正确建立
  - [ ] 数据能正确插入和查询

- [ ] 检查点 2：员工列表管理模块
  - [ ] 能成功保存候选人基本信息（不包含产品线）

  - [ ] 界面交互在员工管理的二级菜单中显示员工列表模块

- [ ] 检查点 3：考试阶段（前置阶段）管理模块

  - [ ] 考试阶段数据能正确显示在考试阶段模块中，自动带出候选人基本信息（不包含产品线）
  - [ ] 考试阶段完成后候选人状态自动更新
  - [ ] 考试阶段模块界面交互友好，布局清晰
  - [ ] 界面交互在员工管理的二级菜单中显示考试阶段模块

- [ ] 检查点 4：面试阶段管理模块
  - [ ] 能成功保存推荐面试信息（包含所有要求字段）并关联到候选人

  - [ ] 能成功保存技术面试（二）信息（包含所有要求字段）并关联到候选人
  - [ ] 能成功保存主管面试信息（包含所有要求字段）并关联到候选人
  - [ ] 能成功保存审批信息（包含所有要求字段）并关联到候选人
  - [ ] 能成功保存Offer信息（包含所有要求字段）并关联到候选人
  - [ ] 能成功保存入职信息（包含所有要求字段）并关联到候选人
  - [ ] 能成功保存离职信息（包含所有要求字段）并关联到候选人
  - [ ] 面试阶段数据能正确显示在面试阶段模块中，自动带出候选人基本信息
  - [ ] 面试阶段完成后候选人状态自动更新
  - [ ] 一个候选人能有多个面试阶段记录，每个对应不同产品线

  - [ ] 自动带出候选人基本信息，但只编辑面试阶段的字段内容

- [ ] 检查点 5：产品线关联管理模块
  - [ ] 能在面试阶段成功关联产品线
  - [ ] 一个候选人能关联多个产品线
  - [ ] 能在不同产品线间切换查看候选人信息
  - [ ] 产品线关联数据正确存储和显示
  - [ ] 即使候选人被某个产品线录取，其他产品线的面试记录仍能保留
  - [ ] 产品线关联和切换界面交互友好，操作便捷

- [ ] 检查点 6：状态管理模块
  - [ ] 能成功推进候选人到下一阶段
  - [ ] 能成功回退候选人到上一阶段
  - [ ] 候选人状态能正确显示在系统中
  - [ ] 状态流转符合业务流程

  - [ ] 系统整体功能完整

- [ ] 检查点 8：文档和部署
  - [ ] 系统使用文档完整，易于理解

  - [ ] 系统能成功部署和运行
