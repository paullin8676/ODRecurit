# 候选录入表格结构调整 - 实施计划

## [ ] 任务 1：调整候选录入表格的列顺序
- **优先级**：P0
- **依赖**：无
- **描述**：
  - 调整候选录入表格的列顺序，使其与员工管理表格一致
  - 列顺序：姓名、性别、手机号、邮箱、身份证号、当前阶段、录入日期、最近操作人、操作
- **验收标准**：AC-1
- **测试需求**：
  - `human-judgment` TR-1.1：候选录入表格的列顺序与员工管理表格一致
  - `human-judgment` TR-1.2：列顺序合理，符合用户使用习惯
- **注意**：保持列宽度与员工管理表格一致

## [ ] 任务 2：添加“最近操作人”列
- **优先级**：P0
- **依赖**：任务 1
- **描述**：
  - 在候选录入表格中添加“最近操作人”列
  - 显示最后操作该候选人的用户姓名
- **验收标准**：AC-2
- **测试需求**：
  - `human-judgment` TR-2.1：候选录入表格显示“最近操作人”列
  - `programmatic` TR-2.2：“最近操作人”列显示正确的用户姓名
- **注意**：确保“最近操作人”列的宽度与员工管理表格一致

## [ ] 任务 3：调整操作列功能
- **优先级**：P0
- **依赖**：任务 1
- **描述**：
  - 调整候选录入表格的操作列功能，使其与员工管理表格一致
  - 操作列功能：编辑、删除、推进、回退（根据阶段显示）
- **验收标准**：AC-3
- **测试需求**：
  - `human-judgment` TR-3.1：候选录入表格的操作列功能与员工管理表格一致
  - `programmatic` TR-3.2：操作列功能正常工作
- **注意**：保持操作列的宽度与员工管理表格一致

## [ ] 任务 4：调整列宽度
- **优先级**：P1
- **依赖**：任务 1, 任务 2, 任务 3
- **描述**：
  - 调整候选录入表格的列宽度，使其与员工管理表格一致
  - 确保列宽度合理，内容显示完整
- **验收标准**：AC-4
- **测试需求**：
  - `human-judgment` TR-4.1：候选录入表格的列宽度与员工管理表格一致
  - `human-judgment` TR-4.2：列宽度合理，内容显示完整
- **注意**：保持表格的美观性和可读性

## [ ] 任务 5：测试系统功能完整性
- **优先级**：P2
- **依赖**：任务 1, 任务 2, 任务 3, 任务 4
- **描述**：
  - 测试候选录入模块的各种功能，确保所有功能正常工作
  - 验证与员工管理表格的一致性
- **验收标准**：AC-5
- **测试需求**：

  - `human-judgment` TR-5.2：系统界面和交互与员工管理表格一致
- **注意**：重点测试表格结构和操作功能
