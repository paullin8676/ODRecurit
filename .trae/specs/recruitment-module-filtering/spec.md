# 招聘管理子模块数据过滤 - 产品需求文档

## 概述
- **摘要**：实现招聘管理下的子模块（候选录入、机考管理、韧测管理、面试管理、员工管理）列表数据获取时，只获取对应模块在阶段配置中配置的阶段的数据，确保数据显示的准确性和相关性。
- **目的**：提高系统的准确性和用户体验，确保每个子模块只显示与该模块相关的阶段数据，避免显示与模块无关的数据。
- **目标用户**：系统管理员、招聘人员、HR 人员。

## 目标
- 候选录入模块只显示阶段配置中为其配置的阶段的数据
- 机考管理模块只显示阶段配置中为其配置的阶段的数据
- 韧测管理模块只显示阶段配置中为其配置的阶段的数据
- 面试管理模块只显示阶段配置中为其配置的阶段的数据
- 员工管理模块只显示阶段配置中为其配置的阶段的数据

## 非目标（范围外）
- 不修改阶段配置的管理功能
- 不修改其他模块的数据获取逻辑
- 不修改核心业务逻辑

## 背景与上下文
- 系统目前已经实现了阶段配置功能，允许为每个模块配置可使用的阶段
- 但子模块在获取数据时，没有根据阶段配置进行过滤，导致显示了与模块无关的数据
- 这会影响用户体验，使数据显示不够准确和相关

## 功能需求
- **FR-1**：候选录入模块只获取阶段配置中为其配置的阶段的数据
- **FR-2**：机考管理模块只获取阶段配置中为其配置的阶段的数据
- **FR-3**：韧测管理模块只获取阶段配置中为其配置的阶段的数据
- **FR-4**：面试管理模块只获取阶段配置中为其配置的阶段的数据
- **FR-5**：员工管理模块只获取阶段配置中为其配置的阶段的数据

## 非功能需求
- **NFR-1**：保持系统性能和响应速度不变
- **NFR-2**：确保系统稳定性，不引入新的 bug
- **NFR-3**：保持代码可维护性，遵循现有代码风格

## 约束
- **技术**：Vue.js 前端框架，Element Plus UI 库，Express.js 后端
- **业务**：不影响现有业务流程和数据
- **依赖**：保持与现有系统的兼容性

## 假设
- 系统已有的阶段配置数据需要保持不变
- 现有的业务逻辑和功能需要保持不变
- 重构后的系统需要与现有系统无缝集成
