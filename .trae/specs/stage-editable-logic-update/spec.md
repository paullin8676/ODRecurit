# 阶段可编辑逻辑更新 - 产品需求文档

## 概述
- **摘要**：更新机考管理、韧测管理、面试管理模块的阶段可编辑逻辑，使配置了多个阶段的模块都能支持编辑、推进和查看操作，而其他阶段的记录只能查看。
- **目的**：解决当前阶段可编辑逻辑的限制，使配置了多个阶段的模块能够更灵活地进行编辑和推进操作。
- **目标用户**：系统管理员和招聘管理人员。

## 目标
- 更新机考管理模块的阶段可编辑逻辑，使配置的所有阶段都能支持编辑、推进和查看操作。
- 更新韧测管理模块的阶段可编辑逻辑，使配置的所有阶段都能支持编辑、推进和查看操作。
- 确保面试管理模块的阶段可编辑逻辑保持一致，支持配置的所有阶段的编辑、推进和查看操作。

## 非目标（超出范围）
- 不修改其他模块的阶段可编辑逻辑。
- 不修改阶段配置的管理功能。
- 不修改系统的其他功能。

## 背景与上下文
- 系统当前的阶段可编辑逻辑只允许配置的第一个阶段进行编辑和推进操作，其他阶段的记录只能查看。
- 对于配置了多个阶段的模块（如机考管理的机考申报和机考完成阶段），这种限制导致用户无法在后续阶段进行编辑和推进操作。
- 面试管理模块已经实现了更灵活的阶段可编辑逻辑，支持配置的所有阶段的编辑、推进和查看操作。

## 功能需求
- **FR-1**：更新机考管理模块的 `isCurrentStage` 函数，使其检查当前阶段是否在配置的所有阶段中，而不是只检查是否是第一个阶段。
- **FR-2**：更新韧测管理模块的 `isCurrentStage` 函数，使其检查当前阶段是否在配置的所有阶段中，而不是只检查是否是第一个阶段。
- **FR-3**：确保面试管理模块的 `isCurrentStage` 函数保持不变，继续支持配置的所有阶段的编辑、推进和查看操作。

## 非功能需求
- **NFR-1**：修改后的阶段可编辑逻辑应与面试管理模块的逻辑保持一致。
- **NFR-2**：修改后的代码应保持清晰、简洁，易于理解和维护。
- **NFR-3**：修改后的功能应在所有浏览器中正常工作，不影响系统的其他功能。

## 约束
- **技术**：Vue.js + Element Plus 前端框架，Express.js + Sequelize 后端框架。
- **业务**：修改应符合系统的业务逻辑，不影响现有的业务流程。
- **依赖**：依赖现有的阶段配置功能，不引入新的依赖。

## 假设
- 系统的阶段配置功能已经正确实现，能够返回模块配置的所有阶段。
- 系统的权限管理功能已经正确实现，能够控制用户的操作权限。

## 验收标准

### AC-1：机考管理模块的阶段可编辑逻辑
- **Given**：机考管理模块配置了多个阶段（如机考申报和机考完成阶段）。
- **When**：用户访问机考管理页面，查看处于配置阶段中的记录。
- **Then**：用户可以对处于配置阶段中的记录进行编辑、推进和查看操作，对处于其他阶段的记录只能进行查看操作。
- **Verification**：`human-judgment`

### AC-2：韧测管理模块的阶段可编辑逻辑
- **Given**：韧测管理模块配置了多个阶段（如韧测申报和韧测完成阶段）。
- **When**：用户访问韧测管理页面，查看处于配置阶段中的记录。
- **Then**：用户可以对处于配置阶段中的记录进行编辑、推进和查看操作，对处于其他阶段的记录只能进行查看操作。
- **Verification**：`human-judgment`

### AC-3：面试管理模块的阶段可编辑逻辑
- **Given**：面试管理模块配置了多个阶段（如推荐面试、资面安排、技术面试等）。
- **When**：用户访问面试管理页面，查看处于配置阶段中的记录。
- **Then**：用户可以对处于配置阶段中的记录进行编辑、推进和查看操作，对处于其他阶段的记录只能进行查看操作。
- **Verification**：`human-judgment`
